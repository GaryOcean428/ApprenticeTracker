**1. Brief Thought Process**  
You need a unified wireframe spec covering every top-level module of the GTO app—showing the global layout (header/sidebar) plus list and detail screens for each module. This will give your designers and developers a clear blueprint.

**2. Chosen Approach**  
- **Global Layout**: Define the fixed header and collapsible sidebar.  
- **Per-Module Screens**: For each navigation item, provide two key wireframes: a **List / Dashboard** view and a **Detail / Form** view.  
- **Consistency**: Use the same layout patterns (filters, tables, forms) across modules to minimize cognitive load.

---

## 5. Additional Explanations  
- **Consistency**: All list views share the same filter + table pattern; detail views use tabs.  
- **Mobile Considerations**: Collapse the sidebar into a hamburger menu; convert tables to cards.  
- **Accessibility**: Ensure all forms and tables are keyboard-navigable and screen-reader friendly.
